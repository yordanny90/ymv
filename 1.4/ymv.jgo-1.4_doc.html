JavaScript Get Object {$jgo}

Descripción

Esta clase se encarga de la recolección de datos desde el documento, el envío de los datos al servidor, la recepción de la respuesta dada por el servidor, y la redistribución de información hacia el documento mediante la utilización de atributos.

    JavaScript

    Código JavaScript que se ejecuta de acuerdo al valor del status de la solicitud ajax.

    Los valores de {status} y sus respectivos atributos pueden ser:

    --nocontent

    timeout:

Nodos de lectura

Son los nodos de los que se obtiene el data para el request.

  • -name

    String

    input | select | textarea

    Estos valores se agregan al formData del request de cualquier ámbito que se ejecute.
    Si el atributo tiene un valor, este se usa como name del valor. De lo contrario se usará el valor del atributo [name]

  • --name

    String

    input | select | textarea

    Estos valores se agregan al formData del request del ámbito que se ejecuta.
    Si el atributo tiene un valor, este se usa como name del valor. De lo contrario se usará el valor del atributo [name]

  • --form

    None

    Todos los input, select y textarea contenidos dentro de este nodo son agregados al formData del request del ámbito que se ejecuta.

Funciones y propiedades

A continuación se detallan las funciones y propiedades de la clase $jgo

  • $jgo

    Function

    Crea un objeto $jgo con las configuraciones que se reciben en los parámetros

    • opts

      String | Object

      Opciones de configuración del objeto $jgo. Si se recibe un String, este utilizará como el nombre del ámbito

      Valores por defecto en $jgo.default_opts

      Nota: Si se cumplen las siguientes condiciones, se retornará un objeto $jgo previamente configurado y guardado:

      • Ser el único parámetro recibido
      • Este parámetro debe ser de tipo String
      • Que exista un objeto $jgo previamente guardado con la llave igual a este parámetro

    • cfg

      String | Object

      Opciones de configuración de $.ajax. Si se recibe un String, este utilizará como la url de la configuración

      Valores por defecto en $jgo.default_cfg

    • save

      Boolean | String

      Si el valor equivale a true, el objeto $jgo se configura y se guarda para su uso porterior

      Si el valor es un string, la llave que se usará para guardar y porteriormente obtener el objeto $jgo, será este valor.
      De lo contrario, se usará el nombre del ámbito en opts

  • $jgo.default_opts

    Object

    Almacena el valor por defecto para el parámetro opts de la función $jgo

    • dom

      String

      "jgo"

      El nombre de dominio determina la primera parte de los atributos de dominio y de ámbito. No se recomienda especificar este dato si no se conoce a profundidad del funcionamiento de $jgo

      Digite un nombre de dominio para los atributos de este documento:

      Ejemplo atributo de dominio: -name

    • ambit

      String

      ""

      El nombre de ámbito determina la segunda parte de los atributos de ámbito. Es obligatorio especificar este dato por cada objeto $jgo

      Digite un nombre de ámbito para los atributos de este documento:

      Ejemplo atributo de ámbito: --name

    • root

      Filtro jQuery

      document

      Define el o los elementos del documento de donde se obtendrán todos los nodos requeridos para el objeto $jgo

    • validate

      Function | Boolean

      False

      Si es false, no realiza la validación de los inputs.
      Si es true, compara la cantidad de inputs con la cantidad de inputs válidos según el filtro.
      Se continua solo si estos valores son iguales

      Si es una función la ejecuta, y si devuelve true se continúa. Recibe como único parámetro al objeto $jgo

    • validFilter

      Filtro jQuery

      Function()...

    • enabledFilter

      Filtro jQuery

      ":enabled"

  • $jgo.default_cfg

    Object

    Almacena el valor por defecto para el parámetro cfg de la función $jgo

    • url

      window.location.href

    • method

      "POST"

    • contentType

      String | Boolean

      False

      Tipo de contenido que se envía al servidor

      En caso de utilizar otro tipo de contenido, el valor recomendado es: "application/x-www-form-urlencoded".
      En este caso debe cambiar el valor de processData a true

    • processData

      Boolean

      False

      Procesar datos: Si es false se guardarán los datos en un objeto FormData

    • beforeSend

      Function

      Function()...

      Se recomienda no reemplazar esta propiedad, debido a que el funcionamiento de $jgo lo requiere

    • complete

      Function

      Function()...

      Se recomienda no reemplazar esta propiedad, debido a que el funcionamiento de $jgo lo requiere
